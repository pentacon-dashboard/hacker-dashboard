# Skill — Demo Script

공모전 심사 데모(8분 이내) 표준 시나리오. 개발 편의용 스킬 — 배포 결과물에는 포함 X.

## 목표 스토리 (8분 구성)

| 시간 | 섹션 | 메시지 |
|---|---|---|
| 0:00–0:30 | 오프닝 | "임의 투자 데이터에서 자동으로 대시보드가 생성됩니다" |
| 0:30–2:00 | 업로드 → 자동 분석 | 혼합 CSV (국내주식+코인) 드래그. Router 가 자산군을 식별하는 로그 표시 |
| 2:00–4:00 | 통합 워치리스트 & 실시간 틱 | Upbit WebSocket LIVE 배지. 숫자가 실시간으로 움직이는 화면 |
| 4:00–5:30 | 포트폴리오 손익 & 환율 정규화 | KRW/USD 혼재 포트폴리오 → 한 화면 통합 |
| 5:30–6:30 | 3단 품질 게이트 | 일부러 이상치를 섞은 CSV 로 게이트가 에러를 잡아내는 모습 |
| 6:30–7:30 | Router 결정 근거 & 아키텍처 | "왜 이 analyzer 를 골랐는가" 토글 → LangGraph 다이어그램 |
| 7:30–8:00 | 마무리 | 기술 스택, 차별화 3가지, 추후 확장 |

## 리허설 체크리스트

- [ ] 시드 CSV 3종을 `demo/` 에 고정 — 발표 현장에서 즉시 드래그 가능
- [ ] 네트워크 장애 대비 녹화본 MP4 (풀HD, 자막 포함) 준비
- [ ] 주 URL + 예비 URL 양쪽 health check 2시간 전 재확인
- [ ] Lighthouse 수치 스크린샷 슬라이드 1장
- [ ] 노트북 디스플레이 1920×1080 에서 레이아웃 깨짐 없음 검증
- [ ] 다크모드로 시연 (발표장 조명 대응)

## 발표 중 금기

- 실시간으로 새 기능 시연 X — 반드시 녹화 or 사전 검증된 시드로만
- 사설 API 키가 노출되는 네트워크 탭 열지 않기
- LLM 응답을 현장에서 기대 — 캐시된 골든 경로만 시연
