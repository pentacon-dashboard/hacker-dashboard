# Image Generation Policy

GPT image generation is always allowed as a worker capability during `/harness-run`.

"Always allowed" means worker may request or use generated raster assets without a separate user approval when a sprint's UI or demo quality benefits from it. It does not mean every iteration should generate images.

## Default Path

- Use the built-in `image_gen` tool by default.
- Use `$imagegen` guidance for prompt shaping, editing, transparency, and project-bound asset handling.
- Do not use CLI/API fallback unless the user explicitly confirms it.
- CLI/API fallback requires `OPENAI_API_KEY`; never ask the user to paste the key in chat.

## Budget

Default budget per iteration:

- maximum 3 final project-bound image assets
- discarded variants do not need to be tracked unless they influenced the final choice
- if more assets are needed, record the reason in `work-*.md`

## Allowed Uses

- landing or hero imagery
- dashboard empty states
- demo visuals
- product/mock UI imagery
- illustrative raster assets
- evidence screenshots generated by the harness

## Prohibited Uses

Do not use generated images to replace:

- financial numbers
- deterministic metrics
- charts that should be rendered from data
- source row evidence
- API or fixture evidence
- PB/client advice support

For investment analysis, generated images are decorative or explanatory only. They are never evidence.

## Required Request File

Before generating a project-bound image, create:

```text
.harness/runs/<run_id>/image-request-<asset_slug>.md
```

Minimum content:

```markdown
# Image Request - <asset_slug>

## Intended Use
<component/page/report>

## Target Path
<path in sprint worktree>

## Prompt
<final prompt>

## Constraints
<must keep / must avoid>

## Budget
<number of final assets>
```

## Paths

Project assets:

```text
frontend/public/generated/<run_id>/<asset-slug>.png
```

Evidence assets:

```text
docs/harness-runs/<timestamp>/assets/<asset-slug>.png
```

Do not leave a project-referenced image only under `$CODEX_HOME/generated_images`.

## Manifest

Update `.harness/runs/<run_id>/image-manifest.json` when images are generated:

```json
{
  "run_id": "20260430-120000-example",
  "assets": [
    {
      "slug": "hero-dashboard",
      "path": "frontend/public/generated/<run_id>/hero-dashboard.png",
      "used_by": "frontend/app/page.tsx",
      "prompt": "string",
      "mode": "built-in image_gen",
      "provenance": "generated by Codex image tool during harness run",
      "final_asset": true
    }
  ]
}
```
